# Overview of Mood-Based Recipe Recommendation System

## SCRUM-299: MVP Phase
- User authentication (signup/login with JWT)
- Mood input interface (text-based input)
- Basic recipe recommendation engine filtering recipes by mood tags
- Recipe browsing and detail viewing
- User profile management including saved recipes and mood history
- Backend with Node.js, Express, MongoDB
- Frontend React app with login and mood-based recipe browsing

## SCRUM-300: Post-MVP Enhancements
- Enhanced recommendation engine using user mood history and preferences
- User recipe rating and saving functionality
- Mood tracking history maintenance
- Social sharing features for recipes
- Advanced filtering options (diet, cuisine filters)
- Performance improvements with caching and horizontal scaling

## SCRUM-301: Future Expansions
- Integration with third-party grocery delivery APIs
- AI-generated custom recipes microservice
- Voice input interface using speech recognition
- Microservices architecture for scalability
- Secure third-party API integrations and audit logging

---

This repository contains the full implementation of the MVP backend and frontend as well as scaffolds and placeholders for future feature expansions described above.
